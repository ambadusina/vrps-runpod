## Offline prepass sigmoid overflow warning fix

- User log showed `RuntimeWarning: overflow encountered in exp` from `offline/yolo26m_birefnet.py::_sigmoid()` during BiRefNet mask postprocessing.
- Root cause: very negative model logits make the naive `1 / (1 + exp(-x))` compute `exp(large_positive)`, which overflows even though the mathematically correct sigmoid result is just near zero.
- Replaced the naive sigmoid with a numerically stable positive/negative branch implementation in:
  - `offline/yolo26m_birefnet.py`
  - `offline/yolo26m_efficientsam.py`
  - `offline/yoloworld_efficientsam.py`
- Added a regression test in `tests/test_yolo26m_birefnet.py` for extreme logits `[-1000, 0, 1000]`.

Verification:

- `python -m pytest tests/test_yolo26m_birefnet.py tests/test_yolo26m_efficientsam.py -q` passed: 23 tests.
- Manual warning probe confirmed all three sigmoid helpers return `[0.0, 0.5, 1.0]` for `[-1000, 0, 1000]` with zero `RuntimeWarning`s.

## HiddenProcess unsigned Windows exit-code overflow fix

- User reported UI offline processing crashed with:
  - `RuntimeWarning: libshiboken: Overflow: Value 4294967295 exceeds limits of type [signed] "int"`
  - `OverflowError`
- Root cause: `HiddenProcess.finished` is a Qt `Signal(int)`, but Windows process exit code `0xFFFFFFFF` was passed through as Python integer `4294967295`, which does not fit Qt's signed 32-bit int.
- Added `_qt_exit_code()` in `ui/services/hidden_process.py` to normalize DWORD-style exit codes into signed 32-bit values before emitting the signal:
  - `0xFFFFFFFF -> -1`
  - `0x80000000 -> -2147483648`
- Added a regression test in `tests/test_process_services.py` that simulates `wait()` returning `0xFFFFFFFF` and verifies `finished` receives `-1`.

Verification:

- `python -m pytest tests/test_process_services.py -q` passed: 6 tests.
- AST syntax check passed for `ui/services/hidden_process.py` and `tests/test_process_services.py`.
- Manual helper probe with Qt runtime paths configured confirmed `4294967295 -> -1`.
