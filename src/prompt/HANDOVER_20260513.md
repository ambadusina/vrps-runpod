# HANDOVER 2026-05-13

## Subtitle Color and Vertical Subtitle Work

- Read `PROJECT.md` and confirmed the app is a local PySide6 settings UI plus
  FastAPI/DLNA realtime passthrough server.
- Added the `SRT subtitle color` selector to the subtitle style page after the
  subtitle direction selector.
- Color presets now map to `subtitle_color`, with the first option
  `Inverse background contrast` storing an empty value so runtime
  `PT_SUBTITLE_COLOR` is omitted and `config.SUBTITLE_COLOR_RAW` remains empty.
- Preview ASS generation now uses the selected SRT color rule, including inverse
  background color when the stored value is empty.
- Runtime subtitle parsing now distinguishes SRT and ASS color behavior:
  - SRT subtitles keep using the global configured subtitle color.
  - ASS subtitles parse `PrimaryColour` and `OutlineColour` from style lines and
    retain the colors per dialogue line where possible.
- Runtime vertical subtitle rendering now keeps original same-time ASS dialogue
  lines as separate vertical columns, preventing Default/Secondary two-line ASS
  subtitles from collapsing into one vertical line.
- Project documentation now notes that `ui/translations/*.json` files must stay
  UTF-8 with BOM and should be read/written with `utf-8-sig`.

## Subtitle Screenshot Debug Tool

- Added `tools/debug_subtitle_screenshots.py` to validate realtime subtitle
  rendering without a DLNA player.
- Default input is `videos/sub_35s.mp4` plus same-stem `.srt` and `.ass`.
- The tool now reads `runtime_cache/ui_settings.json` by default, including
  subtitle direction, mode, color, alpha, v360, FOV, yaw, pitch, and background
  color.
- Green screenshots run RVM matting by default and configure the project GPU
  runtime cache first. Use `--no-matting` only for fast layout-only checks.
- Alpha screenshots split SBS by renderer eye width, then show a fisheye SBS
  layout plus alpha-packed blocks for visual verification.
- The tool samples subtitle cue midpoints and writes four scenario folders under
  `debug_output/subtitle_screenshots`: `green_srt`, `green_ass`, `alpha_srt`,
  and `alpha_ass`, plus `index.json`.
- Verified with `uv run python tools\debug_subtitle_screenshots.py --max-cues 1 --preview-width 640`.
- Fixed SRT vertical rendering by preserving original SRT body lines as separate
  styled lines, so two-line SRT cues render as separate vertical columns instead
  of collapsing/overlapping into one column.

## Home Page Quick Config Collapse

- Reworked the home quick-config area conservatively after rollback.
- `ui/main_window.py` remains compatible with the existing `HOME_HEIGHT` import
  and startup sizing.
- `ui/pages/home_page.py` now replaces the quick-config `QGroupBox` with a
  bordered `QWidget`, a `QToolButton` header, and a content widget.
- The section defaults to expanded. Clicking the header toggles the content
  widget and switches the Qt native arrow between down and right.
- Expanded state keeps the original `HOME_HEIGHT`; collapsed state reduces the
  window height by the quick-config content height.
- Verified with `python -m py_compile ui\pages\home_page.py ui\main_window.py`.
- Refined the collapsed/expanded header behavior:
  - Collapsed title shows only the short quick-config label.
  - Expanded title shows the full restart-server notice.
  - Header button now expands across the whole row so clicking anywhere on the
    row toggles the section.
  - Content area has its own top border and white background to preserve the
    grouped-section visual boundary.
- Fixed home sizing around the project URL row:
  - Home content height now comes from the left panel layout `sizeHint()`
    instead of subtracting the quick-config content height from a fixed number.
  - Main window height includes the status bar height so the bottom project URL
    is not clipped.
- Adjusted the collapsed-height calculation again because the left-panel
  `sizeHint()` kept the window at expanded height. Collapsed mode now subtracts
  the quick-config content height from `HOME_HEIGHT` with a small margin, while
  still adding the status bar height at the main-window level.
- Replaced the collapsed-height estimate with an explicit sum of visible blocks:
  title area, main buttons, quick-config header, project URL, layout margins,
  and spacing. This keeps the window collapsed while preserving the bottom URL
  row.
- Final decision for home quick-config: keep the window height fixed. The quick
  config content still collapses/expands, but `_current_home_height()` always
  returns `HOME_HEIGHT`; the main-window status-bar height compensation remains
  to avoid clipping the project URL.

## Frozen Offline Convert Command

- Fixed packaged offline conversion failure where `offline.convert` used
  `sys.executable tools\offline_passthrough.py ...` inside `pt_core.exe`.
- In a frozen backend, `offline.convert` now calls
  `pt_core.exe tool offline_passthrough ...` or
  `pt_core.exe tool offline_alpha_passthrough ...`.
- Added `main.py` tool dispatch for those two internal tool entry points.
- Fixed the follow-up packaged failure where the tool dispatch called legacy
  tool `main()` functions with an argv parameter. The dispatcher now temporarily
  sets `sys.argv` and restores it after the internal tool exits.
- Forced line-buffered/write-through stdout and stderr for frozen offline/tool
  commands so packaged offline conversion progress logs are emitted while the
  conversion is running instead of flushing only at process exit.
- Verified with:
  - `python -m py_compile main.py offline\convert.py ui\services\process_helpers.py`
  - `uv run python -m unittest tests.test_offline_convert tests.test_main_args`
