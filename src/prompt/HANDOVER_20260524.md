## TensorRT Runtime Download Progress Fix

- User reported the TensorRT runtime download dialog still showed no progress information.
- Root cause:
  - the fallback wheel size is `2,206,065,494` bytes;
  - the dialog progress signal used Qt `Signal(int, int)`, but Qt int is 32-bit signed and overflows above `2,147,483,647`;
  - the first progress emit with `(0, total)` could fail or truncate before reaching the UI.
- Fix:
  - changed the download progress signal to `Signal(object, object)` so Python integers are delivered intact;
  - the dialog now immediately displays `0/total MB` after auto-download starts, using the known fallback size before HTTP headers arrive;
  - the progress bar shows percent text;
  - download finish no longer immediately hides the progress bar by calling a full progress reset.
