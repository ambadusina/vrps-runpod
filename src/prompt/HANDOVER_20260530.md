# HANDOVER 2026-05-30

## Rclone/PikPak media directory startup crash

- User report: after binding a PikPak cloud drive mounted through rclone as a local video folder, the app kept crashing on startup. The traceback showed `Path.resolve()` raising `OSError: [WinError 1005]` inside `media_library.parse_video_dirs()`, reached from `HomePage.update_video_dirs_summary()`.
- Root cause: video directory parsing treated every configured directory as safely resolvable. Some Windows cloud/rclone mounts can fail during `Path.resolve()` even before any actual scan, so a bad configured media root could crash UI initialization and also backend `config.py` import through `PT_VIDEO_DIR`.
- Fix:
  - Added `_safe_resolve()` in `media_library.py` to catch `OSError` and `RuntimeError`.
  - `parse_video_dirs()` now skips unusable configured media directories and falls back to the built-in `videos` directory if all configured paths fail.
  - `build_media_roots()` also skips paths that cannot be resolved, preventing the same class of failure when building labels/roots.
- Behavior after fix: the app can open even if `runtime_cache/ui_settings.json` still contains the rclone/PikPak path. The cloud path is ignored and will not be exposed/processed; other valid local folders continue to work.
