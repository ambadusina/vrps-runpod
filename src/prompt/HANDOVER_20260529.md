## 2026-05-29 DLNA Image Browse Test Switch

User wants to test whether different DLNA players can browse and play still
images.

Implemented an independent image switch, initially default on for testing:

- `PT_DLNA_IMAGE_ENABLED=1` was the initial test default in `config.py`.
- UI startup now exports `PT_DLNA_IMAGE_ENABLED` from `Settings.server_env()`.
- `runtime_cache/ui_settings.json` initially had `dlna_image_enabled=true` for the active
  UI-run configuration.

Behavior added:

- DLNA directory scans include image files when the switch is enabled.
- Supported image extensions: `.jpg`, `.jpeg`, `.png`, `.webp`, `.bmp`, `.gif`.
- JPEG and PNG expose DLNA profile names (`JPEG_LRG`, `PNG_LRG`); other image
  formats use generic `http-get:*:<mime>:*` protocol info.
- DIDL image items use `object.item.imageItem.photo` and `/media/{name}` URLs.
- ConnectionManager `GetProtocolInfo` now advertises image MIME types when the
  image switch is enabled.
- `/media/{name}` can serve enabled image files with real image `Content-Type`
  and existing byte Range support. Passthrough routes still accept only video
  sources.
- ContentDirectory schema version bumped to `ptv8` / UpdateID 8 so clients are
  more likely to refresh the changed Browse shape.

## 2026-05-29 Offline Single Custom End Time

User requested a new single-video offline test duration option: custom end time.

Implemented:

- `ui/pages/offline_page.py` single-video time row now has a new duration combo
  item `custom_end` shown as "Custom end time" / "自定义结束时间".
- Selecting it shows a right-side `HH:MM:SS` end-time input matching the start
  time input width.
- Before starting a single offline conversion, the UI now probes the selected
  video duration and validates:
  - start time format;
  - custom end time format;
  - custom minutes > 0;
  - start time not later than video duration;
  - end time later than start time;
  - end/clip end not later than video duration.
- The offline command still uses the existing `--start` + `--duration` contract;
  custom end time is converted to `duration = end - start`, so no MatAnyone2 or
  offline tool hot path changes were needed.
- Added CN/EN/JA translations for the new option and validation messages.
- Added `tests/test_offline_page_time_range.py` for time parsing, range
  validation, and UI `run_single()` conversion to `--start`/`--duration`;
  extended UI smoke to assert the new combo item/control exists.

## 2026-05-29 Offline Single Output End-Time Filename

User found that custom-end-time outputs still named the segment only with start
and duration, e.g. `*_S000500_75S_*`, making the selected end time hard to see.

Implemented:

- `offline/convert.py::_single_out()` now inserts an end-time tag for any
  positive-duration single output:
  - old: `S000500_75S`
  - new: `S000500_E000615_75S`
  - for start `00:05:00` and end `00:16:15`, new name is
    `S000500_E001615_675S`.
- `utils/offline_outputs.py` generated-output detection now accepts both legacy
  `Sxxxxxx_duration` and new `Sxxxxxx_Exxxxxx_duration` names, so old generated
  files remain recognized.
- Updated offline conversion and output-detection tests for the new naming.
