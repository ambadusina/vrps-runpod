## CuPy CUDA DLL Packaging Fix

- Investigated packaged startup failure where CuPy import raised `FileNotFoundError` for the distribution root `bin` directory.
- Root cause: PyInstaller collected CUDA DLLs under `_internal`, but CuPy 14 on Windows can infer the distribution root as `CUDA_PATH` and call `os.add_dll_directory(<dist>\bin)` during import.
- Added `packaging/runtime_hook_cuda_dlls.py` and wired it into both PyInstaller builds so CUDA/CuPy DLL paths are registered before any CuPy import.
- Updated `build_exe.bat` to create an empty `<dist>\bin\.keep` during packaging. CUDA DLLs remain only under `_internal`, avoiding a large duplicated package.
- Confirmed the codebase uses `cp.RawKernel` heavily, so first-run warmup remains important to catch CuPy/NVRTC JIT packaging issues on clean machines.
- Fixed `cuda.pathfinder` subprocess probes in frozen `pt_core.exe` by forwarding internal module-style invocations from `main.py` through `runpy.run_module()` before normal CLI parsing.
- Added `packaging/hooks/hook-cupy.py` and wired `packaging\hooks` into the server PyInstaller build because the frozen package missed CuPy Cython extensions such as `cupy._core._carray`.
- Strengthened `hook-cupy.py` to recursively collect `cupy/**/*.pyd` and `cupy_backends/**/*.pyd`, and added a post-build check that fails if `_internal\cupy\_core\_carray*.pyd` is absent.
- Explicitly hidden-imported `cuda.pathfinder._dynamic_libs.dynamic_lib_subprocess` and added a post-build runtime probe because CuPy/NVRTC header lookup launches that module in a frozen subprocess. Pure Python modules can live inside PyInstaller's PYZ archive, so the check runs `pt_core.exe -m ...` instead of looking for a physical `.py` file.
- Defaulted `CUPY_COMPILE_WITH_PTX=1` in packaged runtime and GPU cache setup to avoid CUDA 12.6 NVRTC emitting unsupported cubin for Blackwell `sm_120` GPUs, which caused `CUDA_ERROR_NO_BINARY_FOR_GPU`.
- In frozen runtime, force `CUDA_PATH`/`CUDA_HOME` to the packaged distribution root and preserve originals as `PT_ORIGINAL_CUDA_PATH`/`PT_ORIGINAL_CUDA_HOME`, so CuPy prefers bundled CUDA components over stale or broken user system CUDA installs.
- Updated packaging to copy `%CUDA_PATH%\include` into `<dist>\include` and verify `cuda_fp16.h`, making the packaged distribution a minimal CUDA root for CuPy/NVRTC header lookup.

## DLNA Browse SQLite Library Index

- Implemented the agreed SQLite-backed library cache for DLNA directory browsing.
- Added `PT_LIBRARY_INDEX_DB`, defaulting to `runtime_cache/library_index.sqlite`.
- Added `utils/media_index.py`:
  - stores directory entries, directory signatures, and per-video metadata in one SQLite database;
  - each Browse still performs a lightweight stat pass over the requested directory;
  - changed directories are refreshed, unchanged videos reuse cached metadata;
  - metadata cache keys include path key, size, and `mtime_ns`;
  - failed probes are cached briefly and retried after 60 seconds so partially copied files can recover.
- Updated `dlna/content_directory.py` to read directory children through the media index and keep a small in-memory directory item cache keyed by directory signature and passthrough/chapter config.
- Kept child folder `childCount` as a lightweight directory stat count so browsing a parent folder does not ffprobe every file in every child folder.
- Validation:
  - `python -m compileall config.py utils\media_index.py dlna\content_directory.py` passed.
  - Smoke test with a temporary media directory confirmed a second Browse hits the same snapshot and adding `b.mp4` changes the directory signature and returns both files.
