## Offline MatAnyone2 Main10 Prepass Fix
- User reported packaged offline alpha MatAnyone2 medium failed on RTX 4070 Laptop 8GB with an 8K HEVC Main10 source before any YW+ES sample log was produced.
- Root cause: YOLO-World/EfficientSAM prepass read PyNv decoded planes as NV12 unconditionally. Main10 sources decode as P016/P010-like `uint16` planes, so `cv2.cvtColor(..., COLOR_YUV2BGR_NV12)` received `CV_16U` input and failed with OpenCV `Unsupported depth of input image`.
- Added `offline/decoded_frames.py` with `decoded_frame_to_bgr(frame)`. It detects `uint16` decoded planes, right-shifts P016/P010 values to 8-bit NV12, then runs OpenCV NV12-to-BGR conversion.
- Updated `offline/yoloworld_efficientsam.py` to use the shared helper for YOLO-World/EfficientSAM prepass frames.
- Updated both SAM3 prepass callers in `tools/offline_alpha_passthrough.py` and `tools/offline_passthrough.py` to use the same helper, preventing the same Main10 crash in slow mode.
- Verified `uv run python -m py_compile offline\decoded_frames.py offline\yoloworld_efficientsam.py tools\offline_alpha_passthrough.py tools\offline_passthrough.py`.
- Verified the helper with a mocked `uint16` P016-style frame converts to `uint8` BGR without triggering OpenCV `CV_16U` errors.

## RTX 20 AV1 Decode Routing Fix
- User reported realtime and offline failures on RTX 2060 with 8K AV1 8-bit source. The offline alpha log failed in `PyNvSimpleDecoder.frame_at()` with `Codec not supported on this GPU`.
- Root cause: RTX 20/Turing has no AV1 NVDEC support. `utils.video_metadata.select_backend()` allowed AV1 into the PyNv route, and 8K resolution then selected `pynv_hevc`, so decode failed only after the first frame request.
- Updated `utils/video_metadata.py` so AV1 checks the detected NVIDIA compute capability. If compute capability is missing or below `8.6`, AV1 routes to `ffmpeg_fallback` with a clear reason.
- Added a cached helper for the AV1 GPU capability check to avoid repeated `nvidia-smi` probes.
- Verified with monkeypatch checks: simulated RTX 2060 `cc=7.5` returns `ffmpeg_fallback`; simulated RTX 4070 `cc=8.9` still returns `pynv_hevc` for 8K AV1.
- Verified `uv run python -m py_compile utils\video_metadata.py tools\offline_alpha_passthrough.py tools\offline_passthrough.py http_app\routes_media.py pipeline\pynv_stream.py`.
- Note: offline alpha can use the existing FFmpeg decode fallback and still encode through the GPU path. Realtime alpha currently still requires the PyNv NV12 live path, so AV1 realtime alpha on RTX 20 remains unsupported until a realtime alpha FFmpeg-decode fallback is implemented.
