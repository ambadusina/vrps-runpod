## 8K Phase 2 ThreadedDecoder Probe

- The probe:
  - resolves root-relative paths such as `videos/test_8k_2.mp4`;
  - uses the existing `cfr_source_index` policy to reproduce the live CFR source-index sequence;
  - measures current `SimpleDecoder[index]` over the same selected source indices;
  - measures `PyNvVideoCodec.ThreadedDecoder.get_batch_frames()` sequential fetch and source-frame discard;
  - runs a separate hash pass for the first selected frames so CPU hash copies do not pollute throughput timings;
  - writes JSON plus UTF-8 BOM Markdown reports under `baseline/`.
- PyNv API finding:
  - installed PyNvVideoCodec is `2.1.0`;

## 8K Phase 2 ThreadedDecoder Mapping Calibration

- Added `tools/pynv_threaded_mapping_probe.py` to explicitly prove ThreadedDecoder mapping stability.
- Documentation checked:
  - NVIDIA PyNvVideoCodec API Reference says ThreadedDecoder `start_frame` seeks to the nearest keyframe and skips frames until the target frame index;
  - `get_batch_frames(batch_size)` retrieves prefetched decoded frames and `batch_size` must not exceed constructor `buffer_size`;

## 8K Stage 3 Encode Input Lifetime Probe

- Added `tools/pynv_encode_lifetime_probe.py`.
- Purpose:
  - determine whether PyNv/NVENC has fully consumed a GPU NV12 input slot when `Encode()` returns;
  - prove whether a Matter-owned NV12 output slot may be immediately reused or must be retained for a ring distance.
- Probe behavior:
  - creates deterministic solid NV12 GPU frames;
  - encodes them with PyNvVideoCodec;
  - optionally overwrites the just-encoded GPU slot immediately after `Encode()` returns;
  - decodes the encoded stream with FFmpeg back to raw NV12;
  - samples Y values and classifies each decoded frame as expected, overwritten, or unknown.
- Findings:
  - one-slot immediate overwrite is unsafe: encoded output after frame 0 can show the overwrite values;
  - `--post-encode-sync` using CUDA null stream synchronize after `Encode()` still does not make immediate overwrite safe;
  - three-slot delayed reuse passes in the current synthetic tests, including 8K HEVC.
- Important 8K validation command:
  `uv run python tools\pynv_encode_lifetime_probe.py --width 8192 --height 4096 --frames 24 --codec hevc --bitrate 50000000 --gop 60 --progress 6 --slots 3`
- Latest 8K output:
  - `baseline/pynv_encode_lifetime_stage3_20260515_111503_740948.json`
  - `baseline/pynv_encode_lifetime_stage3_20260515_111503_740948.md`
  - result: `ok=True`, corrupt/unknown frames `0`, encode FPS `36.48`.
- Production code update:
  - `pipeline/pynv_stream.py` no longer releases green-path `nv12_slot` immediately after `Encode()`;
  - it keeps recently encoded slots in `pending_nv12_slots`;
  - with default `PASSTHROUGH_NV12_RING_SLOTS=3`, the stream retains the latest two slots and only releases the oldest slot when the pending queue exceeds that distance;
  - pending slots are drained after `EndEncode()` and also in worker `finally`.

  - no more Python/GPU probes should be run in this session until the crash dialogs stop and any remaining crashed `python.exe` processes are closed.
