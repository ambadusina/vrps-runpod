## Short DLNA live item green/alpha mode fix

- Bug: for videos shorter than the live chapter threshold, DLNA exposes live passthrough as direct virtual files instead of virtual chapter folders. When green and alpha were both enabled, some clients could treat both short virtual files as green.
- Root cause: short virtual file ids (`lg_...` / `la_...`) were not recognized by `_id_to_live()` / `BrowseMetadata`; only long-video chapter container ids (`pl_...` / `pla_...`) were mode-aware. A client that re-browsed metadata by item id could lose `mode=alpha`.
- Fix:
  - Added explicit short live item prefixes `lg_` and `la_`.
  - `_id_to_live()` now recognizes both short item ids and long chapter container ids.
  - Short live items now carry `passthrough_mode`.
  - `BrowseMetadata` returns item metadata for short live items, preserving `?mode=green` / `?mode=alpha`.
- Validation:
  - Added tests for distinct short live ids/URLs and alpha metadata.
  - `python -m unittest tests.test_content_directory_modes` passed.
  - `py_compile` passed for `dlna/content_directory.py` and the updated test.

## 2026-05-17 DLNA Alpha Virtual File Mode Fix

- Investigated a report where opening `kckc-001.mp4` alpha mode in SKYBOX produced green-screen output.
- `debug_output/server.log` showed the request arrived with `requested_mode=None`, selected `mode=green`, returned `X-Passthrough-Mode=live-mpegts-green`, and PyNv started with `output_mode=green`.
- Root cause: short videos that do not use live chapter containers are exposed as direct virtual files, and `_passthrough_live_query()` only appended `mode=alpha` when `PT_PASSTHROUGH_OUTPUT_MODE=all`. If the server default was green, the alpha virtual file URL could still resolve to green.
- Changed DLNA passthrough live URL generation to always append explicit `mode=green` or `mode=alpha` for virtual passthrough items.
- Verified `dlna/content_directory.py` with `python -m py_compile`.

## 2026-05-17 DLNA DIDL Cache Bust for Mode URLs

## 2026-05-17 Flat 2D Alpha FOV-Based Output Size

- Corrected the flat 2D alpha output-size rule. The first prototype used one eye size `S=max(src_w, src_h)`, which made a 90-degree flat plane occupy only half of the 180-degree fisheye diameter and reduced effective visible resolution.
- Updated `alpha_output_size()` to compute one-eye size as `ceil(max(src_w, src_h) * 180 / PT_ALPHA_2D_FOV)`, rounded to an even value; full SBS output is `2S x S`.
- With `PT_ALPHA_2D_FOV=90`, `1920x1080` now maps to `7680x3840`; with the current local `PT_ALPHA_2D_FOV=80`, it maps to `8640x4320`.
- Verified `pipeline/alpha_packer.py`, `pipeline/pynv_stream.py`, and `tools/offline_alpha_passthrough.py` with `python -m py_compile`.
