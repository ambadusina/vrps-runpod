  - live route was not intentionally converted to staged pipeline.
- Useful finding:
  - old fixed `tempfile.TemporaryDirectory` monkey patch in the full-chain probe could hang CuPy/RawKernel NV12 composite;
  - probe now uses `configure_gpu_runtime_cache()` and keeps fixed tempdir behind `PT_PYNV_FULLCHAIN_FIXED_TEMPDIR=1`.

  - `baseline/pynv_fullchain_stage4_staged_smoke_20260515.json`;
  - `baseline/pynv_fullchain_stage4_staged_120_20260515.json`;
  - 120-frame steady FPS was around `31.76`, with decode still the bottleneck.
- ThreadedDecoder staged attempt caused native Windows `python.exe` crash dialogs:
  - user reported access violation: instruction `0x00007FFFCCC27880` referenced memory `0x0000000B69200000`; memory could not be read;
  - this is not a catchable Python exception.
- Most likely cause:
  - ThreadedDecoder returned frames were passed across worker threads while decode continued calling `get_batch_frames()`;
  - this violates the previously proven lifetime rule that returned frames must be consumed before the next `get_batch_frames()` call.
- Safety action:
  - `tools/pynv_fullchain_probe.py --decoder threaded` was explicitly disabled with `SystemExit`;
  - do not resume ThreadedDecoder staged work until an isolated ownership probe proves a safe device-to-device copy into owned GPU buffers, or decode+matting are redesigned to remain in the same batch lifetime scope.

## Alpha Passthrough Threaded Decoder Flicker

- User reported a severe visual correctness issue:
  - alpha passthrough with `threaded_serial` can make the red alpha channel flicker continuously on some sources;
  - the same source is stable with the old `simple` decoder;
  - example source: `videos\72456_3840p.mp4`.
- Risk assessment:
  - this matches earlier PyNv ThreadedDecoder lifetime sensitivity;
  - alpha path runs multiple CUDA/ORT steps after decode, so it must not keep reading PyNv-managed batch frame memory directly.
- Fix:
  - `pipeline/pynv_io.py`: added `owned_copy()` for `GpuNv12Frame` and `GpuP016Frame`, copying PyNv-owned planes into CuPy-owned contiguous GPU buffers and synchronizing that handoff;
  - important implementation detail: `owned_copy()` must use `CudaPlane.as_cupy()` rather than `cp.asarray(raw_pyNv_view)` because PyNv's raw CUDA Array Interface may contain list fields that CuPy rejects;
  - owned copy plus pre-copy `cp.cuda.Device().synchronize()` did not eliminate visible flicker, so it is not sufficient as the production answer;
  - `config.py`: added diagnostic flag `PT_PASSTHROUGH_ALPHA_ALLOW_THREADED_DECODER=1`;
  - `pipeline/pynv_stream.py`: realtime alpha now falls back to effective `decoder=simple` when global decoder is `threaded_serial`, unless the diagnostic flag is explicitly enabled;
  - `tools/offline_alpha_passthrough.py`: offline alpha uses the same guarded behavior.
- Scope:
  - green-screen passthrough keeps the configured decoder and can still use `threaded_serial`;
  - simple decoder behavior is unchanged;
  - alpha + threaded remains available only for diagnostics through `PT_PASSTHROUGH_ALPHA_ALLOW_THREADED_DECODER=1`, and still uses `owned_copy()` when enabled.
