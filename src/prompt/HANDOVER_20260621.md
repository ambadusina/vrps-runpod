# HANDOVER 2026-06-21

## SI 改为实时 MPEG-TS 路径 + [SI] 时间索引目录

状态：已落地，设计（两层目录 = 选择时间索引入口 + ≤10 快速章节，对齐
passthrough_live）经用户确认。代码 + 测试就绪（117 passed），仅剩实机播放体验验证。

### 背景 / 动机

- 缓存版 `/media_si`（progressive virtual MP4）首播前要等整条 ~50min 混音 AAC
  编码完才能服务第 0 字节，实测准备 1~3 分钟，无法实时。根因是架构性的：单
  moov + 全量 sample 表必须先有完整音频。
- 现有实时 SI 路径 `http_app/si_stream.py::LiveStreamSession`（源视频 `-c:v copy`
  + SI 混音）此前输出 fMP4，且没有接到任何路由。fMP4 正是 SKYBOX 卡顿的根因
  （见 `tools/si_proto/FINDINGS.md`：fMP4 触发 3.6x~6.35x 重叠 Range 风暴）。
- 关键发现：`/passthrough_live`（透视/绿幕/alpha/2D转3D 实时）输出的是
  **MPEG-TS（`media_type="video/MP2T"`）**，SKYBOX 已验证可播。SI 实时应复用同一
  种线性 TS 传输，而非 fMP4。

### 改动

1. `http_app/si_stream.py`
   - `LiveStreamSession._start_ffmpeg`：输出由 `-f mp4 +frag_keyframe+empty_moov`
     改为 `-f mpegts`（加 `-muxpreload 0 -muxdelay 0`）。视频仍 `-c:v copy`，仅 SI
     混音编码 → 快于实时，无整片缓存。HEVC/H.264 → TS 的 mp4toannexb 由 ffmpeg
     按输入编解码自动插入（不硬编码，保持编解码无关）。
   - 新增模块级 `iter_si_mpegts(video, si_wav, config, start_time)`：每次 spawn 一个
     ffmpeg，流式 yield stdout 到 EOF，结束清理。

2. `http_app/routes_media.py`
   - 新增 `GET /si_live/{name:path}`：接受 `?t=<秒>` 与 `.ts/.m2ts/.mpegts` 后缀提示，
     `StreamingResponse(media_type="video/MP2T")` 实时推 SI 混音流。仅需同名
     `.si.wav`，零缓存，首帧 ~1-2s。seek = 客户端用新的 `?t=` 重新请求（与
     passthrough_live 一致）。
   - 旧的缓存版 `/media_si` HEAD/GET 路由**保留**为休眠 fallback（不在 DLNA 暴露）。

3. `dlna/content_directory.py`
   - `[SI]` 由单 item（指向 `/media_si`）改为 **container（目录）**，前缀 `six_`，
     标题 `[SI]<vr_name>`，**结构完全对齐 passthrough_live**：
     - 浏览 `[SI]`（`_si_chapter_items`）= 1 个「选择时间索引」子目录入口（`sxi_`）
       + 最多 N（默认 10，`PASSTHROUGH_LIVE_CHAPTER_MAX_ITEMS`）条快速直接播放章节
       叶子（`sic_...@offset`，复用 `_live_chapter_offsets`）。container child_count
       = `len(_live_chapter_offsets)+1`。
     - 「选择时间索引」(`sxi_`) 进入完整时间索引树（复用 Live 数学 helper
       `_live_time_group_ranges/_live_time_minute_offsets/_live_time_point_offsets/
       _live_time_index_child_count/_fmt_index_time/_live_time_force_hours`）：
       组 10min `sig_...@start-end` → 分 1min `sin_...@start` → 5s 点叶子 `sit_...@offset`
       （单组时跳过组层，直接列分钟）。
     - 所有播放叶子（章节 + 5s 点）url = `/si_live/<quoted>.ts?t=<offset>&ptv=<ver>`，
       mime `video/MP2T`，dlna_pn `HEVC_TS_NA_ISO`，protocol_info 复用
       `_live_passthrough_protocol_info`；由 `_si_play_leaf` 统一构造。
   - 新增 id 解析 `_id_to_si_dir(six_)/_id_to_si_time_index(sxi_/sig_/sin_)/
     _id_to_si_point(sic_/sit_)` 与 item 构造 `_si_container_item/
     _si_time_index_root_item/_si_chapter_items/_si_time_minute_items/
     _si_time_index_items/_si_time_index_metadata_item/_si_point_metadata_item/
     _metadata_didl_for_si_dir`；`handle_soap` 的 Browse(DirectChildren+Metadata)
     已接线（`six_`→章节列表，`sxi_/sig_/sin_`→时间索引，`sic_/sit_`→播放叶子元数据）。
   - 前缀互不冲突：`six_/sxi_/sig_/sin_/sic_/sit_` 都不以 `si_/sg_/sa_/pl*/li*/img_`
     开头（已用 id 解析 round-trip 验证无交叉匹配）。
   - 删除已死代码：`_si_item_from_video`、`_estimated_si_size`、`_si_protocol_info`、
     `prewarm_progressive_si_virtual_mp4` 导入，以及 dlna 侧 `prewarm_si/
     si_prewarm_remaining/SI_BROWSE_PREWARM_LIMIT` 接线（实时路径不需要预热）。

### 未做 / 注意

- `_DIDL_SCHEMA_VERSION` **未** bump（仍 `ptv10`），以免大面积改动 ptv10 断言。
  代价：客户端若缓存了旧 `si_` item，仍会显示并经保留的 `/media_si` 慢路径播放；
  重新 Browse 即得到新的 `[SI]` 目录。若要强制所有客户端刷新，后续可 bump 到
  `ptv11` 并同步更新测试里所有 `ptv10` 断言。
- `config.py::SI_BROWSE_PREWARM_LIMIT` 仍在（其他地方/测试可能引用），只是 dlna 不再用。
- 缓存版 `/media_si` + 并行分段重混（summary 20260618 计划、R1 接缝一致性）现在
  **不再是 SI 实时的关键路径**，仅与休眠的缓存 fallback 相关。

### 实机待测（下一步，唯一剩余项）

- 目录结构已经用户确认。剩 SKYBOX / nPlayer 上浏览 `[SI]` 目录 → 选快速章节或时间
  索引点 → `/si_live` MPEG-TS 是否秒开、是否流畅、seek 体验是否符合预期（参照
  passthrough_live 的 time-index）。
- 若某播放器对 `?t=` 重启式 seek 不友好，可参考 passthrough_live 的 managed live
  session / 防探测风暴逻辑，将 SI 折进 passthrough_live 的会话机制（当前 `/si_live`
  是独立轻量实现，未带那套客户端专属保护）。
