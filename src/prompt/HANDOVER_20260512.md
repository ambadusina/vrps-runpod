## Alpha passthrough subtitle projection fix

- User reported subtitles do not show on alpha passthrough video and noted alpha mode needs flat subtitle plane -> fisheye projection, unlike green-screen mode.
- Root cause found in `pipeline/pynv_stream.py`:
  - subtitles were applied after `AlphaPacker` had already projected/packed the frame into fisheye alpha layout;
  - that reused the green/SBS subtitle overlay path on the final alpha-packed frame, so the subtitle was not part of the source image before fisheye projection and was not included in the packed alpha mask.
- Updated alpha live path:
  - `PyNvPassthroughStream` now obtains the existing v360 subtitle overlay once per frame;
  - in alpha mode, the overlay is blended into the uploaded original NV12 frame before alpha packing;
  - the same positioned overlay list is passed to `AlphaPacker` so the subtitle region is merged into the alpha mask before fisheye projection and six-block alpha packing;
  - green mode still applies subtitles after green compositing as before.
- Updated `pipeline/alpha_packer.py`:
  - added `merge_overlay_alpha` CUDA kernel;
  - `pack_uploaded()` can accept subtitle overlays and merge their alpha into the low-res alpha mask;
  - `pack_gpu_nv12_frame()` / `pack_gpu_p016_frame()` support a `before_pack` callback that runs after frame upload + alpha inference and before fisheye alpha packing.
- Updated `tests/test_alpha_packer.py`:
  - verifies the `before_pack` callback sees the uploaded NV12 frame;
  - verifies the callback-returned overlay list is passed into `pack_uploaded()`.
- Verification:
  - `python -m unittest tests.test_alpha_packer` passed;
