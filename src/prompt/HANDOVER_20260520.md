## Realtime PyNv Bitrate Source Cap Removal
- User correctly pointed out realtime should not follow the offline `source` bitrate alignment policy.
- Root cause: realtime `pipeline/pynv_stream.py` used `effective_default_bitrate()`, which applies `PASSTHROUGH_HEVC_SOURCE_MAX_MULTIPLIER=2.0` by default. For low-bitrate 2D sources such as the `dance_*.mp4` samples, realtime HEVC could be capped near source bitrate x2 instead of using configured 50M, damaging chroma and making red alpha blocks appear gray after encode/playback.
- Changed PyNv realtime preflight and stream encoder creation to use `_realtime_pynv_bitrate()`, which parses `PT_PASSTHROUGH_HEVC_BITRATE` directly and ignores source bitrate caps. Offline `--bitrate source` behavior remains separate.
- Added `tests/test_pynv_stream_bitrate.py` to lock realtime behavior.
- Verification: `uv run python -m py_compile pipeline\pynv_stream.py tests\test_pynv_stream_bitrate.py`; `uv run python -m unittest tests.test_pynv_stream_bitrate tests.test_content_directory_modes tests.test_offline_outputs`.
